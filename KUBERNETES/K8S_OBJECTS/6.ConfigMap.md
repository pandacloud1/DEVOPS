## ğŸ“¦ Pod Manifest (using ConfigMap as Volume)
File name: pod.yaml
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: mypod
spec:
  containers:
    - name: c1
      image: httpd:latest
      volumeMounts:
        - name: config-volume
          mountPath: /etc/config
  volumes:
    - name: config-volume
      configMap:
        name: my-configmap   # must match the ConfigMap name
```

## Commands
You can read the data stored in configmap, but this is not the case with secrets
```bash
kubectl describe configmap my-configmap
```

You can check the content inside the pod
```bash
kubectl exec -it my-pod -c c1 -- bash
cd /etc/config
```

Delete the configmap & pod
```bash
kubectl delete -f configmap.yaml
```
```bash
kubectl delete -f pod.yaml
```
